# Security Policy

## Reporting a vulnerability

Please use GitHub's private vulnerability reporting for sensitive security issues in this repository.

- Open the repository's **Security** tab and use **Report a vulnerability**.
- Include reproduction steps, impact, and any suggested mitigations if you have them.

Do not open a public issue for an undisclosed vulnerability.

## Non-sensitive problems

Use [GitHub Issues](https://github.com/alxnva/babel/issues) for non-sensitive bugs, broken links, configuration problems, and hardening ideas that do not expose users or infrastructure.

## Disclosure expectations

- Reports will be triaged privately first.
- Fixes may be prepared in private before public disclosure.
- Public follow-up will happen after a fix or mitigation is ready.
